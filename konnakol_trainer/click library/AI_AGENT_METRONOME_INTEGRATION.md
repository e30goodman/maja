# Документация для ИИ-Агента: Интеграция Селектора Звуков

Привет! Это инструкция для ИИ-разработчика, который будет переносить этот компонент выбора звуков в основной проект метронома.

## ⚠️ КРИТИЧЕСКИЕ ОГРАНИЧЕНИЯ: АУДИОДВИЖОК (НЕ ВЛЕЗАТЬ!)

**Крайне важно понимать:** структура тонов и конфигурация аудио (`src/audio/engine.ts` и `src/audio/sounds.ts`) ЖЕСТКО ЗАФИКСИРОВАНЫ и выверены.

В них реализован математически строгий **трехголосный паттерн**:
1. **Первая доля:** Акцент (`accentFreq`)
2. **Вторая и четвертая доля:** Пассивный базовый тон (`baseFreq`)
3. **Третья доля:** Выборочный тон (`altFreq` или `altNoiseFreq`)

❌ **Строго запрещено:**
- Вносить рандомизацию (`Math.random()`) в частоты среза фильтров или тонов.
- Нарушать 3-х голосный паттерн в методе `scheduleNote`.

## 🎵 Интеграция ваших встроенных звуков
В массиве `MOVEMENT_SOUNDS` (в `src/audio/sounds.ts`) вы найдете два пустых пресета:
`id: "classic"` и `id: "old-school"`.
Это выделенные места (заглушки) с пометкой `// TODO`. Вам нужно будет заменить их параметры на ваши оригинальные звуки, которые уже были встроены в проект до этого.

В файлах `engine.ts` и `sounds.ts` оставлены капсом предупреждения-блокировки. Оставьте их на месте!

## 🧩 Подключение UI: Bottom Sheet Selector

Как внедрить этот селектор:

1. **Компонент:** Возьмите компонент `SoundSelectorGrid` (мемоизированную нижнюю шторку) целиком и перекиньте в новый проект.
2. **Кнопка-триггер (Click Sound):**
   ```tsx
   <button 
    onClick={() => setIsSelectorOpen(true)}
    className="bg-[#131722] border border-[#1f2438] hover:bg-[#1a2030] hover:border-[#2a3048] px-3 py-1.5 rounded-lg flex items-center transition-all group"
   >
      <span className="text-[#a4abc5] text-xs font-medium group-hover:text-[#d6d9e6] transition-colors line-clamp-1 text-center">
        {activeSound.name}
      </span>
   </button>
   ```
3. **Обработка закрытия (onClose):**
   Паттерн требует, чтобы при предпросмотре звука из меню, этот "тестовый" звук замолкал при выходе из меню, **но только если** метроном до этого не был запущен.
   ```tsx
   const handleCloseMenu = useCallback(() => {
     setIsSelectorOpen(false);
     if (!isPlaying) {
       clearPreview(); // Сброс таймаута предпросмотра
       engine.stop();
     }
   }, [isPlaying, clearPreview]);
   ```

4. **Автоматическая остановка предпросмотра (1.5 такта):** При клике на звук предпросмотр должен автоматически замолкать ровно через 6 долей (1.5 такта), основываясь на текущем BPM:
   ```tsx
   const handleSelectSound = useCallback((id: string) => {
     // ...
     engine.play(sound);
     if (!isPlaying) {
       clearPreview();
       const msPerBeat = (60 / bpm) * 1000;
       previewTimeoutRef.current = window.setTimeout(() => engine.stop(), msPerBeat * 6); // 6 долей = 1.5 такта
     }
   }, [isPlaying, bpm, clearPreview]);
   ```

5. **Рендер компонента:** Вставьте шторку `SoundSelectorGrid` в JSX в соответствующее место, соблюдая все анимации `motion` (см. App.tsx как референс).

Сетка внутри шторки спроектирована максимально компактно (`grid-cols-4 sm:grid-cols-5`, без нумерации внутри, `p-2`), чтобы на мобильном устройстве помещалось много звуков без скролла. Обводка отвечает за индикацию активного элемента, никаких motion-layout анимаций там нет.
