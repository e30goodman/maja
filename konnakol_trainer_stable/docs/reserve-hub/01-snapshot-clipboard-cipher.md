# 1. Алгоритм «шифра» компактного снепшота (буфер обмена)

## 1.1. Зачем он короткий

Идея не в криптографии, а в **сжатии без потери** для ссылки в мессенджере.

1. **Сетка (акценты, Ta, поддоли, мультипликаторы, pulse-unlink)** упакована в **бинарный блок** фиксированного порядка полей, затем **Base64 URL-safe** (`+`→`-`, `/`→`_`, без padding `=`). Это даёт ~4/3 рост от сырого байта вместо длинной строки из нулей и единиц.
2. **Префикс `p1` / `p2` / `p3`** одним символом версии формата перед Base64; приёмник по префиксу выбирает парсер.
3. **Мёртвые ячейки (dead cells)** кодируются отдельным компактным токеном `encodeDeadCellsToken`, а не JSON-объектом.
4. **Остальное состояние** (темп, число тактов, слогов на такт, chaos, пресет клика) — это **короткие десятичные числа** и один **битовый `flags`**.

Итоговая строка — это **точки как разделители полей**, без ключей JSON → мало символов.

## 1.2. Внешний вид строки (актуальный формат)

Маркер (см. константы в `App.tsx`):

- `SNAPSHOT_CLIPBOARD_MARKER` — человекочитаемый префикс с «ʘᴗʘ♪» (в т.ч. для узнаваемости в чате).
- `SNAPSHOT_CLIPBOARD_MARKER_REGEX` — тот же маркер, но допускает **zero-width** символы (мессенджеры любят вставлять `\u200b` и т.п. между символами).

Тело после маркера — **ровно 8 сегментов**, разделённых **одной** точкой `.` (пробелы в теле вычищаются):

```text
{tempo}.{bars}.{syllables}.{gridToken}.{deadCellsToken}.{chaos}.{flags}.{soundId}
```

Пример структуры (условно):

```text
(ʘᴗʘ)♪:120.4.4.p3AbC....15.123.3
```

- `tempo` — целое BPM, допустимый диапазон при декоде жёстко проверяется.
- `bars`, `syllables` — размеры сетки.
- `gridToken` — см. §1.3.
- `deadCellsToken` — компактное описание «обрезанных» рядов.
- `chaos` — уровень chaos randomizer.
- `flags` — десятичное представление битовой маски `buildSnapshotFlags`.
- `soundId` — индекс в `CLICK_SOUND_PRESET_ORDER` (см. `buildSnapshotSoundId` / `applySnapshotSoundId`).

## 1.3. Внутренность `gridToken` (`packGridTokenPacked` / `unpackGridTokenPacked`)

Строка: **`p1` | `p2` | `p3` + Base64Url(bytes)`**.

Байтовый поток (после декодирования Base64), с **магическим байтом** `0x50` и **версией** `0x01`…`0x03`:

| Порядок | Содержимое |
|---------|------------|
| 1 | `0x50` |
| 2 | версия сетки: `0x01` p1, `0x02` p2, **`0x03` p3 (сейчас пишется всегда)** |
| 3–4 | `bars`, `syllables` (uint8, clamp в encode) |
| 5 | count кастомных слогов по рядам + пары `(row, value)` |
| 6–7 | `cellCount` uint16 BE — число ячеек в линейном индексе `cells` |
| 8 | битовая карта **акцентов** по ячейкам (плотно, по 8 бит на байт) |
| 9* | *только version ≥ 0x03:* битовая карта **Ta-ding** по тем же ячейкам |
| 10 | список **поддолей** ≠1: count uint16 + записи `(idx u16 BE, value u8)` |
| 11 | мультипликаторы по ряду (count + пары) |
| 12 | pulse meter unlinked: список номеров рядов |
| 13 | хвост **accentMapVersion** (логика различается p2 vs p3 — см. комментарии в `unpackGridTokenPacked`; в p3 без хвоста трактуется как явная карта Ta) |

**Почему p3 обязателен в нормальном экспорте:** в коде `useV3 = true` при упаковке — иначе при приёме старых блобов «ломается» белая рамка Ta на первой доле (см. комментарий в `packGridTokenPacked` про байт карты).

## 1.4. Битовые флаги `buildSnapshotFlags` / `applySnapshotFlags`

Каждый флаг — отдельный бит в **одном** числе `flags` в строке:

| Бит | Константа | Смысл |
|-----|-----------|--------|
| 0 | `SNAPSHOT_FLAG_RANDOM_MODE_ENABLED` | Randomizer включён на границах тактов |
| 1 | `SNAPSHOT_FLAG_RANDOM_PULSATION` | Ось Pulse |
| 2 | `SNAPSHOT_FLAG_RANDOM_PATTERN` | Ось Accent/pattern |
| 3 | `SNAPSHOT_FLAG_RANDOM_SPEED` | Ось Speed |
| 4 | `SNAPSHOT_FLAG_RANDOM_BAR_SPEED` | Ось Dead/bar speed |
| 5 | `SNAPSHOT_FLAG_PANEL_EXPANDED` | Панель развёрнута |
| 6 | `SNAPSHOT_FLAG_ONLY_ACCENTS` | Legacy «только акценты» |
| 7 | `SNAPSHOT_FLAG_FIRST_BEAT_ACCENT` | Глобальный Ta / first beat |
| 8 | `SNAPSHOT_FLAG_POLY_MODE` | Полиритм |
| 9 | `SNAPSHOT_FLAG_POLY_VOICES_3` | 3 голоса (иначе 2) |
| 10 | *(зарезервировано)* | 4 голоса — в коде отключено |
| 11 | `SNAPSHOT_FLAG_PARENT_MODE` | Parent random |

**Важно:** полный JSON-слот в `localStorage` (`snapshotToJSON`) хранит больше полей, чем буфер; буфер — **облегчённый обмен**, не дублирует parent-genome и т.д.

## 1.5. Ветки `tryDecodeSnapshotClipboard` (не сломать порядок)

После снятия маркера тело режется по `.`:

1. **11 частей** — старый «расширенный» компакт (отдельные токены акцентов/поддолей/… как в раннем формате). Поддерживается для совместимости.
2. **8 частей** — **актуальный** путь: `gridToken` либо `p*+base64`, либо legacy `accent|syllables|...`, либо один `gridToken` hex для очень старых акцентов + `deadCells`.
3. **7 частей** — ещё более старый вариант **без** dead-cells сегмента.

Плюс префиксы **JSON**:

- `konnakolTrainerSnapshotV1:` — `JSON.parse` → `parseSnapshotRow`.
- `konnakolTrainerSnapshotV2:` — то же.
- `METRONOME_CONFIG:` — legacy compact без kawaii-маркера.

Любое изменение **числа полей** или порядка в `compact.split('.')` ломает обратную совместимость — добавляйте **новую** ветку по длине или новый префикс, не перетирайте старую.

## 1.6. Полиритм и сетка в буфере

Перед `packGridTokenPacked` акценты и Ta **схлопываются в плоский** набор ключей `r-c`:

- `flattenLaneSetMap(accentsByLane)` и то же для `taDingKeysByLane` при `polyMode === true`.

Иначе в блоб попадёт не то, что видит пользователь в полиритме.

## 1.7. Инструкция: что делать, если «опять сломали»

1. **Симптом:** paste не парсится / сетка пустая / Ta на первой доле везде / мессенджер съел маркер.  
   **Действие:** открыть `tryDecodeSnapshotClipboard` и `packGridTokenPacked`, сравнить с этим файлом; прогнать образец строки вручную по шагам `unpackGridTokenPacked`.

2. **Симптом:** маркер не матчится.  
   **Действие:** проверить `SNAPSHOT_CLIPBOARD_MARKER_REGEX`; не удалять `\u200b` и прочие классы из regex без крайней нужды.

3. **Симптом:** только локальные слоты сломаны, буфер жив.  
   **Действие:** смотреть `snapshotToJSON` / `parseSnapshotRow` — другой контракт, не путать с буфером.

4. **Откат кода:**  
   `git log -p -- konnakol_trainer/src/App.tsx` по ключевым словам выше; точечно `git checkout <commit> -- konnakol_trainer/src/App.tsx` (осторожно с остальными правками).

5. **После фикса:** руками проверить **копирование** слота и **вставку** в новый слот + полиритм on/off + `p3` в середине строки.
