# UI Launch Breakage Prevention

Цель: не допускать белого экрана после правок в коде генерации/логгера/импортов.

## Что именно сломалось в этом инциденте

- В `lessonLogger.ts` остался импорт `computePrasaPolicy`.
- В `parentMode.ts` экспорт `computePrasaPolicy` исчез после серии правок и откатов.
- Vite/rollup не смог собрать модульный граф, приложение не смонтировалось в `#root`.
- В браузере это проявилось как белый экран.

## Почему это произошло

- Правки и откаты делались быстро в нескольких связанных файлах.
- Не был выполнен обязательный контроль "импорт ↔ экспорт" сразу после отката.
- Проверка запуска началась с порта/сервера, а не со строгой сборки.

## Обязательный протокол для агента (если затрагиваются экспорты/откаты/рефакторинг)

1. Сразу после правок выполнить `npm run build` в `konnakol_trainer`.
2. Если есть ошибка `is not exported by`:
   - либо восстановить экспорт в исходном модуле,
   - либо убрать/заменить импорт в потребителе.
3. Проверить, что сборка проходит полностью без ошибок.
4. Только после этого запускать `npm run dev` и проверять UI в браузере.
5. Если был белый экран, сначала считать это модульной/рантайм-ошибкой, а не проблемой порта.

## Мини-чеклист перед ответом "починил UI"

- [ ] `npm run build` успешно.
- [ ] Нет ошибок unresolved import/export.
- [ ] Dev-сервер стартует на нужном порту.
- [ ] Корневой модуль UI не падает при загрузке.
