=== Проект — сводка в одном месте (для памяти агента) ===
- Название продукта в UI: BassMastery — Espace Maja; язык интерфейса курса: французский.
- Назначение: веб-курс бас-гитары (модули, уроки, практики), прогресс чеклистов, модалки фото и режимов (modes), опционально чат с Gemini (ключ из env при сборке).
- Публичный деплой: GitHub Pages из ветки main; URL вида https://<user>.github.io/maja/ (base путь /maja/).
- Исходники сайта, которые реально собираются: только каталог maja/ (Vite). Корень репо (App.tsx, корневой index.html) — старый/параллельный шаблон, не путать с продом.
- Главный файл приложения: maja/index.html — массив COURSE_DATA в <script type="module">, renderApp(), разблокировка модулей (mod-1, mod-2 и т.д. по логике), модалки image-modal и mode-modal.
- localStorage: maja_bass_progress_v2 (JSON-массив id выполненных задач); maja_bass_user_images (JSON объект: ключ = id задачи или scheme-<lessonId> и т.д., значение = массив URL/data URL изображений).
- Статика: maja/public/ → в dist без изменений. Изображения курса: maja/public/images/*. Смежное приложение Konnakol: maja/public/konnakol/index.html, с главной ссылка href="konnakol/" (относительно base /maja/).
- Сборка локально: cd maja && npm ci && npm run build; dev: npm run dev (порт 3000, host 0.0.0.0). Переменные: GEMINI_API_KEY в .env у корня загрузки env для Vite (см. vite.config define).
- CI: .github/workflows/deploy.yml — npm ci/build в maja/, артефакт maja/dist, секрет GEMINI_API_KEY.
- Внизу страницы курса: блок WhatsApp (текст про Artem); кнопка перехода на Konnakol без длинного пояснения «к уроку 10».
- Отдельный текстовый дамп контента курса без ссылок: contenu_site_maja_sans_liens.txt (вспомогательный).
- Репозиторий Git: origin https://github.com/e30goodman/maja.git

=== Claude-Mem (thedotmack/claude-mem) — как «делают» с Cursor (по docs) ===
Репо: https://github.com/thedotmack/claude-mem | Доки: https://docs.claude-mem.ai/ | Индекс для ИИ: https://docs.claude-mem.ai/llms.txt

--- Зачем и из чего состоит ---
- Память между сессиями: хуки пишут observations (инструменты, правки файлов, shell и т.д.), worker сжимает/индексирует, контекст подмешивается в новые сессии.
- Worker: HTTP API на порту 37777 (по умолчанию), в браузере — http://localhost:37777 . БД: ~/.claude-mem/claude-mem.db ; настройки: ~/.claude-mem/settings.json ; логи: ~/.claude-mem/logs/
- MCP-сервер (mcp-server.cjs) — тонкая обёртка над worker API: без запущенного worker MCP-запросы не обслуживаются (см. раздел Technical Details в docs «Memory Search»).

--- Установка под Cursor (официальные два пути) ---
Док: https://docs.claude-mem.ai/cursor/index.md

1) Универсально: npx claude-mem install — инсталлятор сам находит IDE, в т.ч. Cursor, копирует плагин, ставит зависимости, поднимает worker (см. Installation).

2) Только Cursor (без подписки Claude Code): клон репо → bun install && bun run build → bun run cursor:setup — мастер настроит бесплатный провайдер (Gemini и т.д.), хуки, worker. Детали: https://docs.claude-mem.ai/cursor/gemini-setup и openrouter-setup.

3) Уже есть Claude Code + плагин: после /plugin install выполнить установку хуков Cursor: claude-mem cursor install (в доке — Path B). Проверка: claude-mem cursor status (или из каталога репо: bun run cursor:status). Хуки должны лежать в .cursor/ (hooks.json). После установки перезапустить Cursor.

Не путать: npm install -g claude-mem — в основном SDK, без хуков и worker.

--- Как взаимодействовать с памятью из чата (MCP, «как в доке») ---
Док: https://docs.claude-mem.ai/usage/search-tools

- Порядок всегда: search (индекс, мало токенов) → при необходимости timeline (якорь или query, depth_before/after) → get_observations(ids=[...]) батчем по отобранным ID.
- __IMPORTANT в MCP — текстовое напоминание этого пайплайна (в доке: «no need to invoke», просто видно агенту).
- Параметры search: query, limit, offset, type / obs_type, project, dateStart/dateEnd, orderBy. Синтаксис запросов — SQLite FTS5 (AND/OR/NOT, фразы в кавычках и т.д., см. ту же страницу).
- Проверка данных без MCP: curl "http://localhost:37777/api/search?query=..."

--- Skills vs «мозг» по корпусу (Claude Code; в Cursor — те же MCP-инструменты) ---
Док: https://docs.claude-mem.ai/usage/knowledge-agents.md

- /mem-search (skill в Claude Code) ≈ живой поиск по БД: трёхшаговый MCP search/timeline/get_observations — когда нужны конкретные наблюдения и ID.
- /knowledge-agent ≈ срез истории в корпус и диалог с ним: build_corpus(...) → prime_corpus(name) → query_corpus(name, question) ; при уходе темы — reprime_corpus(name) ; после новых observations — rebuild_corpus(name) затем снова prime/reprime.
- Под капотом те же операции доступны через POST /api/corpus* на worker (таблица в конце страницы Knowledge Agents).

--- Проверка «всё живо» (из Cursor Integration doc) ---
- worker: bun run worker:status (или аналог из установленного пакета)
- хуки Cursor: bun run cursor:status
- UI памяти: открыть http://localhost:37777

--- Если MCP в Cursor UI включён, а агент получает fetch failed ---
- Частая причина по архитектуре: worker на :37777 не запущен / недоступен из среды, где крутится воркер агента; либо MCP-процесс не проксируется до localhost. Смотреть worker:status и логи в ~/.claude-mem/logs/ .
